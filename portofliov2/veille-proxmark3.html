Qu'est-ce que le Proxmark3 ?

Le hardware

Le Proxmark3 est un dispositif matériel conçu par Jonathan Westhues en 2007 et maintenu par la communauté open-source (fork Iceman). Il embarque un FPGA (Xilinx XC3S100E) et un ARM (AT91SAM7S256) qui gèrent les antennes LF et HF en temps réel.

La version PM3 Easy est la plus accessible — clone économique parfait pour débuter.

Usages principaux

  • Lire et analyser des cartes RFID/NFC inconnues
  • Cloner des badges d'accès (tests autorisés)
  • Émuler des cartes sans-contact
  • Cracker les clés Mifare Classic (MFOC / MFCUK)
  • Sniffer les échanges lecteur ↔ carte
  • CTF et challenges de sécurité physique
c
⚠️

Avertissement légal — L'utilisation du Proxmark3 sur des systèmes sans autorisation explicite est illégale. Cette documentation est à but éducatif et de recherche uniquement. Toujours obtenir un accord écrit avant tout test sur des systèmes réels.

LF vs HF — Les deux fréquences

Basse fréquence
125 kHz

Portée ~10 cm. Protocoles simples, peu sécurisés. Très courants dans les badges d'accès anciens.

EM4100 / EM410x HID Prox Awid Indala FDX-B Keri
Haute fréquence
13.56 MHz

Portée ~10 cm (NFC). Protocoles plus complexes, avec chiffrement. Utilisés dans les cartes bancaires, passeports, transports.

Mifare Classic Mifare DESFire iCLASS ISO 14443A ISO 15693 NTAG
ℹ️

Mifare Classic (1K/4K) est la cible la plus fréquente en pentest physique — ses clés A/B 48 bits peuvent être cassées par attaque nested (MFOC) ou dark-side attack (MFCUK) avec le Proxmark3.

Installation sur Linux

1

Installer les dépendances

Sous Debian / Ubuntu / Kali Linux. Ces paquets sont nécessaires pour compiler le firmware et le client.

bash — dépendances 
$ sudo apt update # mise à jour des dépôts

$ sudo apt install -y git build-essential pkg-config \
  libreadline-dev libssl-dev libbz2-dev \
  gcc-arm-none-eabi libnewlib-dev \
  libqt5gui5 libqt5widgets5 qt5-default

$ arm-none-eabi-gcc --version
arm-none-eabi-gcc (15:10.3-2021.07-4) 10.3.1 20210621
[+] toolchain ok
2

Cloner le dépôt Iceman (fork officiel)

Le fork Iceman est le plus actif et le mieux maintenu. Il inclut des améliorations majeures par rapport au code original.

bash — clone 
$ git clone https://github.com/rfidresearchgroup/proxmark3.git
cloning into 'proxmark3'...
remote: enumerating objects: 148423, done.
receiving objects: 100% (148423/148423), done.

$ cd proxmark3
$ ls
armsrc/  client/  common/  doc/  firmware/  tools/  makefile
3

Compiler le firmware et le client

La compilation génère le firmware (flashé sur le Proxmark) et le client (interface en ligne de commande).

bash — compilation 
# pour pm3 easy (clone générique)
$ make platform=pm3generic

# pour un proxmark3 rdv4 original
$ make platform=pm3rdv4

# attendre ~5 min... puis :
  cc      fpga_compress.c
  link    proxmark3
[=] done compiling proxmark3 client
💡

Sur un PM3 Easy, utilise toujours PLATFORM=PM3GENERIC. Le mauvais paramètre peut bricker l'appareil.

4

Configurer les droits USB

Sans cette étape, tu devras lancer le client en root à chaque fois.

bash — permissions 
$ sudo make accessrights
adding udev rules for proxmark3...
[+] udev rules installed

# ajouter l'utilisateur au groupe dialout
$ sudo usermod -ag dialout $user

# appliquer sans redémarrer
$ newgrp dialout
5

flasher le firmware et lancer le client

connecte le proxmark3 en usb, puis flash le firmware compilé. à faire une seule fois (ou après une mise à jour).

bash — flash + lancement 
# vérifier que l'appareil est détecté
$ ls /dev/ttyacm*
/dev/ttyacm0

# flasher le firmware
$ ./pm3-flash-all
waiting for proxmark3 to appear on /dev/ttyacm0
flashing bootrom...
flashing fullimage...
[+] all done. proxmark3 flashed successfully.

# lancer le client
$ ./pm3

Utilisation — Commandes essentielles

Vérifier l'appareil

Première commande à taper pour vérifier que le Proxmark répond correctement.

proxmark3 — hw version 
[proxmark3] pm3 --> hw version

 [ proxmark3 rfid instrument ]
  firmware........................................
  firmware version: iceman/master/v4.17582-1423-g...
  client version..................................
  client: rrg/iceman proxmark3 client v4.17582
  chip: at91sam7s256 rev b
  memory: 256k flash, 64k ram
  fpga: lf/hf fpga image

Détecter une carte basse fréquence (LF)

Approche une carte ou un badge LF de l'antenne gauche.

proxmark3 — lf search 
[proxmark3] pm3 --> lf search

[=] note: some demods may fail or not trigger...
[=] checking for known tags...

[+] em 410x id 1122334455
[+] em410x - found!
   em tag id    : 1122334455
   dez 8        : 03730485
   dez 10       : 0289740885
[+] possible de-scramble pattern

Détecter une carte haute fréquence (HF)

Approche une carte NFC / badge Mifare de l'antenne droite.

proxmark3 — hf search 
[proxmark3] pm3 --> hf search

[=] searching for iso14443-a tag...
[+] uid: 04 a2 b3 c4 d5 e6 f7
[+] atqa: 00 04
[+] sak: 08 [2]
[=] possible types:
    mifare classic 1k
[=] prng type: weak
[+] valid iso 14443-a tag found

Attaque Mifare Classic — autopwn

La commande hf mf autopwn lance automatiquement plusieurs attaques (nested, hardnested, darkside) pour récupérer toutes les clés d'une carte Mifare Classic et dump le contenu.

1

Lancer autopwn

proxmark3 — autopwn 
[proxmark3] pm3 --> hf mf autopwn

[=] mifare classic autopwn started
[=] checking for default keys...
[+] found valid key [ a ] block 00: ffffffffffff
[=] nested attack...
[+] found key [ b ] block 00: a0a1a2a3a4a5
[+] found key [ a ] block 04: d3f7d3f7d3f7
[+] found key [ b ] block 04: d3f7d3f7d3f7
...
[+] all keys found!
2

Dump complet de la carte

proxmark3 — dump 
[proxmark3] pm3 --> hf mf dump --1k -f backup_carte.bin

[=] reading sector access bits...
[=] reading 64 blocks...
[+] saved 1024 bytes to file backup_carte.bin
[+] saved 1024 bytes to file backup_carte.eml
[+] saved to backup_carte.json
3

Écrire le dump sur une carte vierge

proxmark3 — restore 
# carte vierge "magic" (uid modifiable) requise
[proxmark3] pm3 --> hf mf restore --1k -f backup_carte.bin

[=] restoring card from file backup_carte.bin
[=] writing 64 blocks...
[+] card successfully restored
⚠️

Carte magic — pour cloner l'UID, il faut une carte "magic gen1" ou "gen2". Les cartes standards ne permettent pas de modifier le bloc 0 (UID).

📋

Cloner un badge LF EM4100

proxmark3 — clone LF 
# lire le badge source
[proxmark3] pm3 --> lf em 410x reader
[+] em 410x id: 1122334455
   em tag id    : 1122334455

# écrire sur un tag t5577 vierge
[proxmark3] pm3 --> lf em 410x clone --id 1122334455
[=] cloning id 1122334455 to t5577 tag
[+] done. verify with: lf em 410x reader

# vérification
[proxmark3] pm3 --> lf em 410x reader
[+] em 410x id: 1122334455
[+] clone successful!

Référence rapide des commandes

Commande Description
hw version Afficher la version du firmware et du hardware
hw tune Mesurer les fréquences des antennes (calibration)
lf search Recherche automatique de tags LF connus
hf search Recherche automatique de tags HF connus
lf em 410x reader Lire un tag EM4100 (badge LF classique)
lf em 410x clone --id <ID> Cloner un tag EM4100 sur un T5577 vierge
hf mf reader Lire les blocs Mifare Classic avec les clés connues
hf mf autopwn Crack automatique de toutes les clés Mifare Classic
hf mf dump --1k -f <file> Sauvegarder le contenu d'une carte Mifare 1K
hf mf restore --1k -f <file> Restaurer un dump sur une carte magic vierge
hf mf chk --1k Tester les clés par défaut sur Mifare 1K
hf 14a reader Lire les informations d'un tag ISO 14443A
lf sniff Sniffer la communication entre lecteur et carte LF
hf sniff Sniffer la communication HF (capture passive)
script run <script> Exécuter un script Lua d'automatisation
